# ระบบจองห้องประชุม SPK OFFICE (Google Apps Script)

เว็บแอปสำหรับบันทึกการจองห้องประชุมลง Google Sheets โดยตรง

## โครงสร้างไฟล์

- `code.gs` — โค้ดฝั่งเซิร์ฟเวอร์ (อ่าน/เขียน Google Sheets, คำนวณเวลาว่าง, จองซ้ำ, ยกเลิก)
- `index.html` — หน้าเว็บฝั่งผู้ใช้ (ธีมสีไล่โทนแบบ Instagram, ฟอนต์ Kanit, แท็บ Book/Cancel)

## วิธีติดตั้ง

1. เปิด Google Sheet ที่ต้องการใช้เก็บข้อมูลการจอง
2. ไปที่เมนู **ส่วนขยาย (Extensions) > Apps Script**
3. ลบโค้ดเริ่มต้นในไฟล์ `Code.gs` แล้ววางเนื้อหาจาก `code.gs` ในโปรเจกต์นี้แทน
4. สร้างไฟล์ HTML ใหม่ในโปรเจกต์ Apps Script ตั้งชื่อว่า `index` (ไม่ต้องใส่ `.html`) แล้ววางเนื้อหาจาก `index.html`
5. กด **Deploy > New deployment**
   - Select type: **Web app**
   - Execute as: **Me**
   - Who has access: **Anyone** (หรือปรับตามนโยบายองค์กร)
6. กด Deploy แล้วเปิดลิงก์ Web app ที่ได้เพื่อใช้งาน

เมื่อรันครั้งแรก ระบบจะสร้างชีทชื่อ **"Reservations"** พร้อมหัวตารางให้อัตโนมัติ หากยังไม่มีอยู่ในไฟล์ Google Sheets

> UI ทั้งหมดใช้ภาษาอังกฤษ (ตามที่แก้ไว้บน branch)

## Logic การเช็คเวลาว่าง

- Slot เวลาทั้งหมดแบ่งทุก 30 นาที ตั้งแต่ 08:30 ถึง 17:30
- เมื่อผู้ใช้เลือก **ห้อง + วันที่** ระบบจะดึงรายการจองทั้งหมดของห้องนั้นในวันนั้นจากชีท แล้วตัด slot ที่ถูกจองไปแล้วออกจาก dropdown "เวลาเริ่มจอง"
- เมื่อผู้ใช้เลือก **เวลาเริ่มจอง** ระบบจะคำนวณ dropdown "เวลาจบ" โดยจำกัดไม่ให้เลยเวลาที่มีการจองถัดไป (หรือ 17:30 เป็นอย่างช้าที่สุด)
- ก่อนบันทึกจริง ระบบจะตรวจสอบการทับซ้อนของเวลาอีกครั้งฝั่งเซิร์ฟเวอร์ (กัน race condition ด้วย `LockService`) ก่อนเขียนแถวใหม่ลงชีท — ถ้าชนกับที่มีคนจองไปแล้ว (เช่นแทรกจองพร้อมกัน) จะไม่บันทึกซ้ำ และแจ้งผลผ่าน popup ให้ผู้ใช้ทราบทันที

## กันจองซ้ำ + Popup แจ้งผล

- ทุกครั้งที่จอง (สำเร็จหรือไม่สำเร็จ) หน้าเว็บจะเด้ง popup แจ้งผลกลางจอ พร้อมข้อความเหตุผล (เช่น "ช่วงเวลานี้ถูกจองไปแล้ว กรุณาเลือกเวลาอื่น") แทนข้อความเล็กๆ ใต้ฟอร์ม
- ถ้าจองไม่สำเร็จเพราะมีคนจองแทรกไปก่อน ระบบจะรีเฟรช dropdown เวลาว่างให้อัตโนมัติ เพื่อให้ผู้ใช้เลือกเวลาใหม่ได้ทันที

## แสดงตารางการจองที่มีอยู่ (ให้ทุกคนเห็น)

- เมื่อเลือก **ห้อง + วันที่** ระบบจะแสดงรายการ "Already booked" ของห้องนั้นในวันนั้น (เวลาเริ่ม–จบ และชื่อผู้จอง) ไว้ในฟอร์ม เพื่อให้ทุกคนที่เข้ามาใช้งานเห็นคิวที่ถูกจองไปแล้วก่อนเลือกเวลา
- ข้อมูลนี้ดึงจากฟังก์ชัน `getExistingBookings(date, room)` ใน `code.gs` ซึ่งอ่านจากชีท `Reservations` โดยตรงแบบ real-time ทุกครั้งที่เปลี่ยนห้องหรือวันที่

## จองซ้ำแบบ Repeat

- ในฟอร์มจองมีตัวเลือก **Repeat**: No repeat / Daily / Weekly (same day) พร้อมช่อง "Number of times (including first)"
- เมื่อเลือกซ้ำ ระบบ (`submitReservation`) จะสร้างรายการจองหลายวันตามรูปแบบและจำนวนที่เลือก โดย**ตรวจการทับซ้อนของแต่ละวันแยกกัน** — วันไหนว่างก็จอง วันไหนชนก็ข้าม แล้วสรุปผลกลับมา (เช่น "booked successfully: 3 booking(s). (Skipped 1 already-booked date(s): ...)")
- จำกัดสูงสุด 52 ครั้งต่อการจองหนึ่งครั้ง (ตัวแปร `MAX_REPEAT`)

## เครื่องดื่ม / อุปกรณ์ที่ขอใช้

- ในฟอร์มจองมีช่อง checkbox **"Beverage / Equipment Needed (เครื่องดื่ม / อุปกรณ์ที่ขอใช้)"** ให้ผู้จองเลือกได้ตามต้องการ (ไม่บังคับ): **น้ำดื่ม**, **Dongle**
- รายการตัวเลือกกำหนดไว้ในตัวแปร `EQUIPMENT_OPTIONS` ใน `code.gs` — เพิ่ม/ลบ/แก้ตัวเลือกได้โดยแก้อาร์เรย์นี้ ฝั่งหน้าเว็บจะดึงรายการมาแสดงอัตโนมัติผ่าน `getEquipmentOptions()`
- สิ่งที่เลือกจะถูกบันทึกลงคอลัมน์ **Equipment** ในชีท `Reservations`

## ยกเลิกการจอง

- หน้าเว็บมีแท็บ **"Cancel Booking"** ให้ผู้ใช้กรอก **เบอร์ติดต่อที่ใช้จอง (Tel)** เพื่อค้นหาการจองในอนาคต (วันนี้เป็นต้นไป) ของตนเอง แล้วกดยกเลิกทีละรายการ พร้อม popup ยืนยันก่อนลบจริง
- ฝั่งเซิร์ฟเวอร์: `getBookingsByPhone(phone)` คืนรายการจองพร้อม `BookingID`, และ `cancelReservation(id, phone)` ลบแถวตามรหัส โดย**ตรวจเบอร์ติดต่อให้ตรงกันก่อน**เพื่อกันการยกเลิกของคนอื่น (ทำงานภายใต้ `LockService`)

## ติดตั้งเป็นแอปบนหน้าจอหลักมือถือ (Add to Home Screen)

`index.html` ฝัง web app manifest, ไอคอนแอป และ meta tag สำหรับมือถือไว้ในตัว (เป็น data URI ทั้งหมด ไม่ต้องเพิ่มไฟล์แยก) ทำให้หลัง deploy แล้วเปิดลิงก์ Web app บนมือถือ สามารถกด **"เพิ่มไปยังหน้าจอโฮม" (Add to Home Screen)** ได้ทั้ง iOS Safari และ Android Chrome — จะได้ไอคอนแอปสีไล่โทน SPK บนหน้าจอ กดเปิดแล้วแสดงผลแบบเต็มจอไม่มีแถบที่อยู่เว็บ (บน iOS) เหมือนเปิดแอปจริง

**ข้อจำกัดที่ควรทราบ:** Google Apps Script Web App มีแค่ endpoint เดียว (`doGet`) ไม่สามารถเสิร์ฟไฟล์ static แยก (เช่น service worker `.js`) ที่ same-origin ได้ จึงยังไม่รองรับการทำงานแบบออฟไลน์ (offline caching) เหมือน PWA เต็มรูปแบบ — ใช้งานได้เฉพาะไอคอน + เปิดแบบเต็มจอเท่านั้น ถ้าต้องการแอปที่ติดตั้งจาก Store จริง (Google Play / App Store) พร้อมออฟไลน์โหมด จะต้องพัฒนาเป็นแอป native/WebView แยกต่างหาก ซึ่งต้องใช้เครื่องมือพัฒนาและบัญชีนักพัฒนาของ Store นั้นๆ เพิ่มเติม
